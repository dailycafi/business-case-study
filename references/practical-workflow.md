# Practical Development Workflow and Self-Assessment

## 14-Step Development Workflow (4 Phases, 7–8 Weeks)

### Phase 1: Material Digestion (1–2 weeks)

**Step 1: Build a Material Map (素材地图)**
- Read all collected materials (interviews, public info, research)
- Tag each piece: which discussion question could it support?
- Identify information-rich zones and information deserts
- Output: annotated material inventory

**Step 2: Lock Teaching Objectives and Discussion Questions**
- Design 4–6 learning objectives (SMART principle)
- Design 4–6 discussion questions (What→Why→How logic)
- Ensure one-to-one correspondence between objectives and questions
- Output: teaching framework document

**Step 3: Reverse-Engineer Information Needs (信息需求倒推)**
- For each question, list the key information points students need
- Cross-reference with material map: what's available vs. what's missing?
- Classify gaps as "fatal" (case cannot work without it) vs. "nice to have"
- Output: information gap analysis, supplemental interview/research plan

### Phase 2: Case Body Writing (2–3 weeks)

**Step 4: First Draft — Content First, Not Polish**
- Write following the three-part structure (Decision Frame Cut)
- Focus on getting all information in the right sections
- Don't worry about elegant prose yet
- Output: complete first draft

**Step 5: Round 1 Revision — Content Adjustment**
- Run information coverage matrix check
- Add missing information, remove obvious redundancy
- Verify every section supports at least one discussion question
- Check chronological accuracy and data consistency
- Output: content-adjusted draft

**Step 6: Round 2 Revision — Language Polishing**
- Switch to narrative style: add dialogue, specific details, data contrasts
- Run forbidden expressions check (see writing-style.md)
- Ensure objective, neutral tone throughout
- Add time anchors at key nodes
- Output: polished draft

**Step 7: Round 3 Revision — Trimming**
- Check word count against target (Chinese ≤10,000 / English ≤6,000)
- Cut content that doesn't serve any discussion question
- Tighten paragraphs: aim for 150–250 chars (CN) / 100–150 words (EN)
- Finalize Exhibits (5–15, each with number, title, source)
- Output: submission-ready case body

### Phase 3: Teaching Note (2–3 weeks)

**Step 8: Teaching Note Skeleton**
- Write synopsis, learning objectives, intended audiences, assignment questions
- Draft teaching plan with time allocation
- Outline board design
- Output: teaching note skeleton

**Step 9: Question Analysis — Deep Dive**
- For each question, write the 5-part analysis:
  1. Analytical tool introduction (200–300 words)
  2. Analysis process with guiding questions (500–800 words)
  3. Reference answer with theory + case evidence (800–1200 words)
  4. Classroom discussion management suggestions (200–300 words)
  5. Theoretical elevation (200–300 words)
- Output: complete question analysis section

**Step 10: Explicit Elevation Search (升维探索)**
- Review Q3–Q4 analysis: is there a pattern that transcends this case?
- Check: can two theories be creatively integrated?
- Check: does the case challenge or extend existing theory?
- If found: develop the elevation with visualization
- If not found: don't force it — move on
- Output: elevation section (if applicable)

**Step 11: Complete Teaching Note**
- Add suggested readings (10–15 items, APA format)
- Write discussion summary (500–800 words)
- Add case update section (if available)
- Finalize board design with visual elements
- Add supplementary materials list
- Output: complete teaching note

### Phase 4: Self-Assessment (1 week)

**Step 12: Rubric Self-Assessment — Case Body**
- Score each of the 4 dimensions (25 points each)
- Any dimension below "Good" (20 points) → targeted revision needed
- Output: case body score sheet with improvement notes

**Step 13: Rubric Self-Assessment — Teaching Note**
- Score each of the 4 dimensions (25+15+35+25)
- Any dimension below "Good" → targeted revision needed
- Output: teaching note score sheet with improvement notes

**Step 14: Final Integration Check**
- Re-run information coverage matrix
- Verify body-TN alignment (wording consistency, theory citations match)
- Final format compliance check
- Output: submission-ready package

## Quick Self-Check Checklist (5-Minute Speed Check)

### Core Quality (5 items)
- [ ] Decision point is clear: who, when, what choice
- [ ] At least 3 layers of information depth (surface → deep → essential)
- [ ] No obvious "correct answer" — genuine debate space exists
- [ ] Key data points all have sources
- [ ] Narrative style maintained (no consulting jargon, no subjective judgment)

### Teaching Design (5 items)
- [ ] 4–6 discussion questions with clear What→Why→How logic chain
- [ ] Each question has at least one supporting theory/framework
- [ ] Question analysis includes classroom management suggestions
- [ ] Board design covers all discussion questions
- [ ] Teaching plan has realistic time allocation

### Bonus Items (2 items)
- [ ] At least one "elevation" opportunity identified and developed
- [ ] Instructor's disciplinary background aligns well with case topic

## Scoring Reference

| Grade | Total Score | Meaning |
|-------|-----------|---------|
| Excellent | 85+ | Award-competitive quality |
| Good | 75–84 | Publishable with minor revision |
| Passing | 70–74 | Needs targeted improvement |
| Needs Improvement | <70 | Major structural issues |

Score each dimension separately. **Any single dimension below "Good" (20 points on a 25-point dimension, or equivalent) requires targeted revision before submission.**

---

## Red-Line Issues (Hard Fail)

These issues, if present, override the scoring — fix them regardless of total score:

1. **No clear decision point** — case reads as a company history, not a decision scenario
2. **Obvious "correct answer"** — no real debate space, discussion will be one-sided
3. **Fabricated or unverified data** — any key data point without a traceable source
4. **Subjective judgment in case body** — author tells reader what to think ("This was a brilliant strategy")
5. **Teaching note questions disconnected** — questions are independent topics, not a logical arc

## Award Encouragement Directions

Case awards especially value submissions in these three categories:

### Category 1: Frontier Management Topics
- Issues not yet widely discussed in existing case libraries
- Emerging phenomena that may reshape business practices
- Examples: AI governance, platform ecosystem transitions, cross-border digital operations

### Category 2: New Perspectives and Methods
- Classic problems approached from novel theoretical angles
- Cross-disciplinary integration (e.g., behavioral economics + strategy)
- New analytical frameworks developed from case evidence

### Category 3: Novel Teaching Methods
- Innovative classroom delivery approaches
- Creative use of multimedia, simulation, role-play
- Teaching designs that produce unusually high student engagement

---

## 5 Key Reminders

1. **Don't mechanically apply "no redundancy, no gaps"** — narrative completeness and flow always take priority over precise information-to-question mapping
2. **Questions can iterate** — as writing progresses, you may discover better questions; don't lock them prematurely
3. **Don't chase bleeding-edge theory** — a new application of a classic framework is more valuable than citing an unvalidated working paper
4. **Elevation is a bonus, not a requirement** — if it doesn't emerge naturally from the case, move on
5. **The rubric is a reference tool, not shackles** (Rubric是参考工具，不是镣铐) — stay flexible, allow trial-and-error in the creative process
