# Teaching Note Structure and Writing Principles

## Appendix A: 7 Teaching Note Writing Principles

Distilled from case revision practice. Each principle has a verification method (检验方法).

### Principle 1: Theory Is a Knife, Not Decoration

Theory's role is to "cut" the phenomenon — to reveal structures invisible to the naked eye. If removing a theoretical citation does not weaken the analysis, that citation is decorative.

**Verification:** For each theory referenced, ask: "What does this theory reveal that common sense does not?" If no clear answer, the theory is padding.

### Principle 2: The Question Stem Is the Most Important Sentence

The question stem determines everything: what students analyze, which data they focus on, and what frameworks they reach for. A vague question stem produces vague discussion.

**Verification:** Read only the question stems (without the analysis). Can a colleague reconstruct the intended discussion flow? If not, the stems need rewriting.

### Principle 3: Four Questions Are a System

The questions must form a coherent analytical arc, not a list of independent topics. Each question's conclusion becomes the foundation for the next.

**Verification:** Remove any one question. Does the arc collapse or leave a logical gap? If it still works fine, that question may be redundant or disconnected.

### Principle 4: Tension Comes from Analytical Structure, Not Rhetoric

Good case discussion generates tension through the structure of analysis — opposing frameworks, conflicting data, trade-offs with no easy resolution. This tension should not rely on dramatic language.

**Verification:** Strip all adjectives and emphatic language from the teaching note. Does the analytical tension survive? If yes, the structure is doing its job.

### Principle 5: Every Analysis Layer Needs a Classroom Entry Point

Each layer of analysis must have a concrete way for the instructor to enter the discussion: a specific question to ask, a data point to highlight, or a student exercise to run.

**Verification:** For each analytical point, can you write a specific classroom prompt ("Ask students to compare Exhibit 3 and Exhibit 7...")? If not, the analysis is too abstract for classroom use.

### Principle 6: Summaries Are Hinges Between Questions

The summary at the end of each question's analysis is not a conclusion — it is a bridge that pivots toward the next question. It should create momentum, not closure.

**Verification:** Read each question's closing summary. Does it naturally raise the next question? Or does it feel like a period at the end of a paragraph?

### Principle 7: The Title Is a Promise

The case title and teaching note title set expectations for what the reader/instructor will get. A generic title ("Company X's Strategic Transformation") promises nothing. A specific title promises a specific intellectual journey.

**Verification:** Does the title tell a colleague: (1) what company, (2) what tension, (3) what makes this case unique? If any element is missing, the title underperforms.

## Appendix B: 6 Case Body Structural Writing Principles

### Principle 1: Narrative Structure Should Parallel Analysis Structure

The order in which information appears in the case body should mirror the order in which students will analyze it. If Q1 asks about strategic evolution, that information should come before Q3's organizational analysis.

**Verification:** Overlay the information coverage matrix on the body's table of contents. Does the diagonal flow downward-right (earlier sections support earlier questions)? Misalignment signals structural problems.

### Principle 2: The Body Presents Facts; the Teaching Note Explains Them

The case body provides "productive ambiguity" — enough information for analysis but without telling students what it means. The teaching note is where meaning is made explicit.

**Verification:** Search the case body for explanatory sentences ("This shows that...", "The reason was...", "This was a wise/poor decision..."). Every such sentence is a candidate for deletion — move the explanation to the teaching note.

### Principle 3: Tension Through Juxtaposition of Facts, Not Rhetoric

Tension in a case comes from placing contradictory facts next to each other, not from dramatic language. "Revenue grew 344% over 8 years. But 2023 growth plummeted to 15%." — the data creates the tension.

**Verification:** Identify every point of tension in the case. Is it created by data/facts or by adjectives/adverbs? Replace rhetorical tension with factual tension wherever possible.

### Principle 4: Information Giving and Withholding Is a Design Act

Every piece of information in the case is a deliberate choice: what to include, what to omit, what to put in the body vs. Exhibits, what to present prominently vs. bury in a footnote. This is not accidental — it shapes the difficulty and direction of student analysis.

**Verification:** For each Exhibit, ask: "What happens to the discussion if this Exhibit is removed?" If nothing changes, the Exhibit may be unnecessary. If the discussion collapses, the Exhibit is load-bearing.

### Principle 5: Timeline Is the Backbone of the Causal Chain

A clear chronological structure helps students trace cause and effect. Even in a Decision Frame Cut (which opens at the decision point and then goes back), the background section should maintain chronological order internally.

**Verification:** Can a student draw a complete timeline from the case body alone, with all key dates and events? If dates are vague ("recently", "in the past few years"), the causal chain is weakened.

### Principle 6: The Case Ending Should Create an "Unfinished" Feeling

The case should stop at the decision point, not after it. The ending should leave students with the question "What would I do?" — not "What happened next?" If the outcome is revealed, there is nothing left to discuss.

**Verification:** Read the last paragraph. Does it pose a question (explicitly or implicitly)? Or does it resolve the tension? A resolved ending kills classroom discussion.

## Visualization Types for Teaching Notes

When designing frameworks, elevation diagrams, or board plans, choose from these 6 types based on what you need to show:

| Type | Best For | Example |
|------|----------|---------|
| **Matrix (2x2, 3x3)** | Classifying along two dimensions | Ansoff matrix, BCG matrix |
| **Flowchart / Evolution diagram** | Sequential stages, causal chains | Pukang's 3-stage strategic evolution (→ arrows) |
| **Hierarchy (pyramid, concentric circles)** | Levels of abstraction, priority | Bloom's taxonomy pyramid |
| **Comparison table** | Side-by-side evaluation | Dynamic capabilities vs. data network effects |
| **Network diagram (nodes + edges)** | Ecosystem relationships, value networks | Pukang's value network evolution (3 diagrams) |
| **Coordinate axis (X-Y)** | Trade-offs, positioning | Innovation adoption curve |

**Pukang value network visualization example:**
Three network diagrams side by side showing strategic evolution:
```
Stage 1: Isolated node    Stage 2: Connections added    Stage 3: Center node
    ○                         ○──○                        ○──●──○
                              │                           │  │  │
                              ○                           ○──○──○
```
Each diagram makes the strategic shift visible at a glance — far more powerful than describing it in text.
