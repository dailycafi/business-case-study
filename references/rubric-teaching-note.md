# Teaching Note Assessment Rubric (100 Points)

## Dimension 1: Learning Objectives and Question Design (25 Points)

**Award criterion:** Clear, appropriate learning objectives for target course/students; discussion questions with logical chain aligned to objectives.

### Excellent (23тАУ25)

**Learning objectives (4тАУ6):**
- One-to-one correspondence with discussion questions
- SMART principle applied
- Bloom's taxonomy progression from low to high order

**Bloom's levels in question design:**

| Level | Type | Verbs | Example |
|-------|------|-------|---------|
| L1 Remember | Recall | list, describe, identify | "What AI application steps did POOK take?" |
| L2 Understand | Comprehend | explain, summarize, classify | "Explain the Amoeba organization's characteristics" |
| L3 Apply | Use | apply, implement, demonstrate | "Apply value chain analysis to Pukang's strategic evolution" |
| L4 Analyze | Examine | analyze, compare, test | "Compare pros/cons of different org structures" |
| L5 Evaluate | Judge | assess, justify, argue | "Evaluate POOK's AI externalization strategy" |
| L6 Create | Design | design, construct, propose | "Design an ecosystem governance plan for Pukang" |

**Question logic chain тАФ standard WhatтЖТWhyтЖТHow:**

```
Q1 [What]: Fact sorting, phenomenon identification
тФФтФА "What happened? What key decisions were made?"

Q2 [Why]: Causal analysis, deep drivers
тФФтФА "Why did this happen? What are the underlying reasons?"

Q3 [How-1]: Methods, capability identification
тФФтФА "How was it achieved? What methods/capabilities were used?"

Q4 [How-2]: Decision evaluation, future action
тФФтФА "What should be done? How to choose going forward?"

(Optional) Q5 [So What]: Elevation, universal insights
тФФтФА "What implications for other companies? What general principles?"
```

**Example (Pukang case):**
```
Q1 [What]: What strategic transformations did Pukang undergo?
         тЖТ Sort three stages: digitization, full-chain, ecosystem

Q2 [Why]: What org structures matched each strategic transformation?
         тЖТ Analyze org-strategy alignment

Q3 [How]: What capabilities supported this strategic-org alignment?
         тЖТ Identify dynamic capabilities (Sensing-Seizing-Transforming)

Q4 [So What]: How to evaluate Pukang's two strategies (brand vs. ecosystem)?
            тЖТ Open decision + elevation: from dynamic capabilities to data network effects
```

**Special cases:** WhatтЖТWhyтЖТHow order can be adjusted, but logic chain must be complete and transitions natural.

**Question openness check:**
- [ ] No single "correct answer"
- [ ] Can spark collision of viewpoints
- [ ] Requires synthesizing multiple information sources
- [ ] Involves trade-offs
- [ ] Highly relevant to real business situations

### Good (20тАУ22): Objectives basically clear but less measurable; some logic chain but could be tighter
### Passing (15тАУ19): Overly broad objectives; unclear logic chain; single-level questions
### Needs Improvement (0тАУ14): Missing/inappropriate objectives; unconnected questions; too simple/closed

## Dimension 2: Teaching Plan (15 Points)

**Award criterion:** Clear, feasible teaching plan with reasonable time allocation; board design is a plus.

### Excellent (14тАУ15)

**Standard time allocation (90тАУ120 min):**

| Segment | Time | Content | Objective |
|---------|------|---------|-----------|
| Pre-class | - | Students read case, prepare questions | - |
| Warm-up | 5тАУ10 min | Background intro, spark interest | Set context |
| Q1 Discussion | 20тАУ25 min | Fact sorting, framework building | Obj 1 |
| Q2 Discussion | 20тАУ25 min | Deep analysis, causal relationships | Obj 2 |
| Q3 Discussion | 20тАУ25 min | Theory extraction, capability identification | Obj 3 |
| Q4 Discussion | 25тАУ30 min | Integrated decision, open discussion | Obj 4 |
| Summary | 10тАУ15 min | Key insights, theoretical elevation | All |

**Teaching methods:**
1. Small group discussion (10тАУ15 min): 3тАУ5 per group, prepare group position
2. Full class discussion (30тАУ40 min): group presentations, instructor facilitation
3. Theory lecture (15тАУ20 min): introduce frameworks, connect to case specifics
4. Summary (10 min): review insights, connect to course objectives

**Board design principles:**
- Clear logical main line showing discussion progression
- Distinct visual hierarchy with colors, symbols, diagrams
- Keywords distilled: 3тАУ5 per discussion point
- Sufficient blank space for adding student viewpoints

## Dimension 3: Question Analysis (35 Points)

**Award criterion:** Uses appropriate analytical methods/frameworks/theories; guides students through inductive discussion to discover generalizable insights.

### Excellent (32тАУ35)

**Standard analysis structure per question:**
```
[Question X Analysis]

I. Analytical Tool Introduction (200тАУ300 words)
тФЬтФА Framework core content
тФЬтФА Applicable scenarios
тФФтФА Relevance to this case

II. Analysis Process (500тАУ800 words)
тФЬтФА Guiding Q1: help students identify key facts
тФЬтФА Guiding Q2: help students build logical connections
тФЬтФА Guiding Q3: help students apply theoretical framework
тФФтФА Guiding Q4: help students reach conclusions

III. Reference Answer (800тАУ1200 words)
тФЬтФА Main point 1: theory + case evidence
тФЬтФА Main point 2: theory + case evidence
тФЬтФА Main point 3: theory + case evidence
тФФтФА Possible alternative viewpoints with supporting evidence

IV. Classroom Discussion Management (200тАУ300 words)
тФЬтФА How to introduce the discussion
тФЬтФА Likely student viewpoints
тФЬтФА How to handle disagreements
тФФтФА How to transition to next question

V. Theoretical Elevation (200тАУ300 words)
тФЬтФА From case to general principles
тФЬтФА Implications for other companies
тФФтФА Connection to other course content
```

**Theory source requirements:**
- Academic frontier: top journals in last 5 years (SMJ, AMJ, ASQ, MS, etc.)
- Classic theory: foundational research (Porter, Teece, Barney, etc.)
- Practice wisdom: benchmark company best practices
- Cross-disciplinary: integrate strategy, organization, technology management

**Analysis quality checklist:**
- [ ] Each question has clear theoretical framework support
- [ ] Theory explanation is clear and student-accessible
- [ ] Specific case evidence supports analysis
- [ ] Logical rigor in reasoning
- [ ] Multiple possible analysis paths presented
- [ ] No single "standard answer" тАФ encourages critical thinking
- [ ] Includes specific discussion facilitation suggestions
- [ ] Rises to generalizable principles and insights

### Bonus: Discussion Question "Elevation" (хНЗч╗┤)

**What is elevation:** Rising from specific case to general principles; jumping from single theory to cross-disciplinary integration; developing new analytical frameworks from existing theories.

**Three paths to elevation:**

**Path 1: Theoretical Integration**
Example (POOK): Integrate Innovation Adoption Curve + Kotter's Change Model тЖТ "Balls in Boxes" visualization method. Map 5 adopter roles into 8 change steps.

**Path 2: Theoretical Extension**
Example (Pukang): Extend Porter's Value Chain тЖТ Value Network Evolution Model (embed in network тЖТ add connections тЖТ move to center). Visualized as three network diagrams.

**Path 3: Create New Framework**
Example (POOK): R.I.C.H. framework for AI implementation's human dimension:
- R: Responsibility (leadership)
- I: Inertia (organizational)
- C: Challenge (employee)
- H: Humanity (community of shared interests)

**Elevation opportunity checklist:**
- [ ] Does the case involve 2+ classic theories? тЖТ Possible integration
- [ ] Does the case phenomenon exceed existing theory's explanatory range? тЖТ Possible extension
- [ ] Does the case reveal a pattern or regularity? тЖТ Possible new analytical tool
- [ ] Can the new framework be visualized? (matrices, flowcharts, pyramids)
- [ ] Is it transferable to other cases?

**Important:** Elevation is "icing on the cake," not forced. Must be grounded in case reality. Not every case can elevate тАФ don't force it, but always watch for opportunities.

### Good (28тАУ31): Frameworks basically fitting but not precise; mostly complete; some elevation
### Passing (21тАУ27): Traditional/broad frameworks; incomplete analysis; minimal elevation
### Needs Improvement (0тАУ20): Inappropriate/missing frameworks; shallow analysis; no elevation

## Dimension 4: Writing and Information Support (25 Points)

**Award criterion:** Well-structured, accurate, readable; valuable summary; post-case updates if available; supplementary materials for students and instructors.

### Excellent (23тАУ25)

**Standard teaching note structure:**
```
1. Synopsis (300тАУ500 words)
2. Learning Objectives (4тАУ6 items, SMART)
3. Intended Audiences and Courses
4. Assignment Questions (4тАУ6 questions)
5. Suggested Readings (10тАУ15 items, APA format)
   - Academic papers: 3тАУ5 (recent 5 years)
   - Classic works: 2тАУ3
   - Business cases: 2тАУ3
   - Industry reports: 2тАУ3
   - Media articles: 2тАУ3
6. Teaching Plan (time allocation + methods)
7. Case Discussion and Analysis (core section)
8. Discussion Summary (500тАУ800 words)
9. Case Update (300тАУ500 words, if available)
10. Board Design
11. Appendix (if needed)
```

**Reading material quality:**
- [ ] All references have complete APA citations
- [ ] Academic papers from SSCI/SCI journals
- [ ] At least 50% published in last 5 years
- [ ] Highly relevant to case discussion themes
- [ ] Cover different disciplinary perspectives
- [ ] Include both Chinese and English materials (if applicable)

**Discussion summary structure:**
1. Core insights (3тАУ4 items): key findings, implications, theory connections
2. Theoretical contribution: how case validates or challenges existing theory
3. Practical implications: specific advice for managers
4. Teaching value: case's unique position in the course

**Supplementary materials (when possible):**
- Video: company promotional, founder interviews, product demos
- PPT: background intro (10тАУ15 slides), theory framework lecture (15тАУ20 slides)
- Data files: Excel financial models, industry data
- Teaching aids: role-play guides, group discussion prompt cards

### Good (20тАУ22): Mostly complete structure; some summary; limited supplementary materials
### Passing (15тАУ19): Incomplete structure; thin summary; missing readings
### Needs Improvement (0тАУ14): Chaotic/incomplete; no summary; severely lacking
