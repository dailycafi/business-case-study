# Teaching Note Assessment Rubric (100 Points)

## Dimension 1: Learning Objectives and Question Design (25 Points)

**Award criterion:** Clear, appropriate learning objectives for target course/students; discussion questions with logical chain aligned to objectives.

### Excellent (23–25)

**Learning objectives (4–6):**
- One-to-one correspondence with discussion questions
- SMART principle applied
- Bloom's taxonomy progression from low to high order

**SMART principle example (POOK case):**
```
1. Explore the key steps enterprises should take when introducing and
   implementing AI technology.                              [Specific, Measurable]
2. Apply the Task-based Approach to understand how AI can be integrated
   into business processes.                                 [Achievable]
3. Discuss the adaptability of different organizational types when
   introducing AI.                                          [Relevant]
4. Anticipate the broad impact of LLMs as a general-purpose technology.
                                                            [Time-bound]
```

**Objective layer design (3–4 objectives):**
```
Layer 1: Knowledge & Comprehension
- Understand the background of...
- Describe the key features of...

Layer 2: Application & Analysis
- Apply [framework] to analyze...
- Identify the key factors in...

Layer 3: Synthesis & Evaluation
- Evaluate the pros/cons of...
- Propose improvements for...
```

**Bloom's levels in question design:**

| Level | Type | Verbs | Example |
|-------|------|-------|---------|
| L1 Remember | Recall | list, describe, identify | "What AI application steps did POOK take?" |
| L2 Understand | Comprehend | explain, summarize, classify | "Explain the Amoeba organization's characteristics" |
| L3 Apply | Use | apply, implement, demonstrate | "Apply value chain analysis to Pukang's strategic evolution" |
| L4 Analyze | Examine | analyze, compare, test | "Compare pros/cons of different org structures" |
| L5 Evaluate | Judge | assess, justify, argue | "Evaluate POOK's AI externalization strategy" |
| L6 Create | Design | design, construct, propose | "Design an ecosystem governance plan for Pukang" |

**Question logic chain — standard What→Why→How:**

```
Q1 [What]: Fact sorting, phenomenon identification
└─ "What happened? What key decisions were made?"

Q2 [Why]: Causal analysis, deep drivers
└─ "Why did this happen? What are the underlying reasons?"

Q3 [How-1]: Methods, capability identification
└─ "How was it achieved? What methods/capabilities were used?"

Q4 [How-2]: Decision evaluation, future action
└─ "What should be done? How to choose going forward?"

(Optional) Q5 [So What]: Elevation, universal insights
└─ "What implications for other companies? What general principles?"
```

**Example (Pukang case):**
```
Q1 [What]: What strategic transformations did Pukang undergo?
         → Sort three stages: digitization, full-chain, ecosystem

Q2 [Why]: What org structures matched each strategic transformation?
         → Analyze org-strategy alignment

Q3 [How]: What capabilities supported this strategic-org alignment?
         → Identify dynamic capabilities (Sensing-Seizing-Transforming)

Q4 [So What]: How to evaluate Pukang's two strategies (brand vs. ecosystem)?
            → Open decision + elevation: from dynamic capabilities to data network effects
```

**Special cases:** What→Why→How order can be adjusted, but logic chain must be complete and transitions natural.

**Question openness check:**
- [ ] No single "correct answer"
- [ ] Can spark collision of viewpoints
- [ ] Requires synthesizing multiple information sources
- [ ] Involves trade-offs
- [ ] Highly relevant to real business situations

### Good (20–22): Objectives basically clear but less measurable; some logic chain but could be tighter
### Passing (15–19): Overly broad objectives; unclear logic chain; single-level questions
### Needs Improvement (0–14): Missing/inappropriate objectives; unconnected questions; too simple/closed

## Dimension 2: Teaching Plan (15 Points)

**Award criterion:** Clear, feasible teaching plan with reasonable time allocation; board design is a plus.

### Excellent (14–15)

**Standard time allocation (90–120 min):**

| Segment | Time | Content | Objective |
|---------|------|---------|-----------|
| Pre-class | - | Students read case, prepare questions | - |
| Warm-up | 5–10 min | Background intro, spark interest | Set context |
| Q1 Discussion | 20–25 min | Fact sorting, framework building | Obj 1 |
| Q2 Discussion | 20–25 min | Deep analysis, causal relationships | Obj 2 |
| Q3 Discussion | 20–25 min | Theory extraction, capability identification | Obj 3 |
| Q4 Discussion | 25–30 min | Integrated decision, open discussion | Obj 4 |
| Summary | 10–15 min | Key insights, theoretical elevation | All |

**Teaching methods:**
1. Small group discussion (10–15 min): 3–5 per group, prepare group position
2. Full class discussion (30–40 min): group presentations, instructor facilitation
3. Theory lecture (15–20 min): introduce frameworks, connect to case specifics
4. Summary (10 min): review insights, connect to course objectives

**Board design principles:**
- Clear logical main line showing discussion progression
- Distinct visual hierarchy with colors, symbols, diagrams
- Keywords distilled: 3–5 per discussion point
- Sufficient blank space for adding student viewpoints

**Board design example (Pukang):**
```
┌─────────────────────────────────────────────────┐
│  Q1: Three Stages of Strategic Transformation   │
│  ┌──────────┬──────────┬──────────┐             │
│  │2010-2016 │2017-2022 │2023-     │             │
│  │Online    │Full-chain│Ecosystem │             │
│  └──────────┴──────────┴──────────┘             │
│                                                  │
│  Q2: Org Structure Evolution                     │
│  Amoeba ──→ Amoeba+Middle ──→ Platform          │
│  [pros]      [pros]            [pros]            │
│  [cons]      [cons]            [cons]            │
│                                                  │
│  Q3: Dynamic Capabilities                        │
│  Sensing ──→ Seizing ──→ Transforming            │
│                                                  │
│  Q4: Future Strategic Choice                     │
│  [Own Brand] ↔ [Ecosystem Scale]                 │
└─────────────────────────────────────────────────┘
```

### Good (12–13): Time allocation basically reasonable; board exists but not clear enough
### Passing (9–11): Unreasonable time allocation; board missing or too simple
### Needs Improvement (0–8): No teaching plan; no board design

## Dimension 3: Question Analysis (35 Points)

**Award criterion:** Uses appropriate analytical methods/frameworks/theories; guides students through inductive discussion to discover generalizable insights.

### Excellent (32–35)

**Standard analysis structure per question:**
```
[Question X Analysis]

I. Analytical Tool Introduction (200–300 words)
├─ Framework core content
├─ Applicable scenarios
└─ Relevance to this case

II. Analysis Process (500–800 words)
├─ Guiding Q1: help students identify key facts
├─ Guiding Q2: help students build logical connections
├─ Guiding Q3: help students apply theoretical framework
└─ Guiding Q4: help students reach conclusions

III. Reference Answer (800–1200 words)
├─ Main point 1: theory + case evidence
├─ Main point 2: theory + case evidence
├─ Main point 3: theory + case evidence
└─ Possible alternative viewpoints with supporting evidence

IV. Classroom Discussion Management (200–300 words)
├─ How to introduce the discussion
├─ Likely student viewpoints
├─ How to handle disagreements
└─ How to transition to next question

V. Theoretical Elevation (200–300 words)
├─ From case to general principles
├─ Implications for other companies
└─ Connection to other course content
```

**Framework-question mapping example (POOK case):**
```
Q1 (AI adoption steps):     Innovation Adoption Cycle + Kotter's 8-Step Change Model
Q2 (AI-business integration): Task-based Approach + Value Chain Analysis
Q3 (Org adaptability):      Amoeba Management Model + Modular Organization
Q4 (LLM as GPT):            General-Purpose Technology Theory + Data Network Effects
```

**Advanced technique: Comparison analysis**

Create structured comparison tables to help students see contrasts:
```
Dynamic Capabilities vs. Data Network Effects

| Dimension              | Dynamic Capabilities    | Data Network Effects     |
|------------------------|------------------------|--------------------------|
| Theoretical core       | Sense-Seize-Transform  | Data flywheel             |
| Core approach          | Organizational learning | Platform data accumulation|
| Specific practices     | Internal restructuring  | Ecosystem data sharing    |
| Competitive advantage  | Adaptive capability     | Scale-driven lock-in      |
```

**Theory source requirements:**
- Academic frontier: top journals in last 5 years (SMJ, AMJ, ASQ, MS, etc.)
- Classic theory: foundational research (Porter, Teece, Barney, etc.)
- Practice wisdom: benchmark company best practices
- Cross-disciplinary: integrate strategy, organization, technology management

**Analysis quality checklist:**
- [ ] Each question has clear theoretical framework support
- [ ] Theory explanation is clear and student-accessible
- [ ] Specific case evidence supports analysis
- [ ] Logical rigor in reasoning
- [ ] Multiple possible analysis paths presented
- [ ] No single "standard answer" — encourages critical thinking
- [ ] Includes specific discussion facilitation suggestions
- [ ] Rises to generalizable principles and insights

### Bonus: Discussion Question "Elevation" (升维)

**What is elevation:** Rising from specific case to general principles; jumping from single theory to cross-disciplinary integration; developing new analytical frameworks from existing theories.

**Three paths to elevation:**

**Path 1: Theoretical Integration**
Example (POOK): Integrate Innovation Adoption Curve + Kotter's Change Model → "Balls in Boxes" visualization method. Map 5 adopter roles into 8 change steps.

**Path 2: Theoretical Extension**
Example (Pukang): Extend Porter's Value Chain → Value Network Evolution Model (embed in network → add connections → move to center). Visualized as three network diagrams.

**Path 3: Create New Framework**
Example (POOK): R.I.C.H. framework for AI implementation's human dimension:
- R: Responsibility (leadership)
- I: Inertia (organizational)
- C: Challenge (employee)
- H: Humanity (community of shared interests)

**Elevation opportunity checklist:**
- [ ] Does the case involve 2+ classic theories? → Possible integration
- [ ] Does the case phenomenon exceed existing theory's explanatory range? → Possible extension
- [ ] Does the case reveal a pattern or regularity? → Possible new analytical tool
- [ ] Can the new framework be visualized? (matrices, flowcharts, pyramids)
- [ ] Is it transferable to other cases?

**Important:** Elevation is "icing on the cake," not forced. Must be grounded in case reality. Not every case can elevate — don't force it, but always watch for opportunities.

### Good (28–31): Frameworks basically fitting but not precise; mostly complete; some elevation
### Passing (21–27): Traditional/broad frameworks; incomplete analysis; minimal elevation
### Needs Improvement (0–20): Inappropriate/missing frameworks; shallow analysis; no elevation

## Dimension 4: Writing and Information Support (25 Points)

**Award criterion:** Well-structured, accurate, readable; valuable summary; post-case updates if available; supplementary materials for students and instructors.

### Excellent (23–25)

**Standard teaching note structure:**
```
1. Synopsis (300–500 words)
2. Learning Objectives (4–6 items, SMART)
3. Intended Audiences and Courses
4. Assignment Questions (4–6 questions)
5. Suggested Readings (10–15 items, APA format)
   - Academic papers: 3–5 (recent 5 years)
   - Classic works: 2–3
   - Business cases: 2–3
   - Industry reports: 2–3
   - Media articles: 2–3
6. Teaching Plan (time allocation + methods)
7. Case Discussion and Analysis (core section)
8. Discussion Summary (500–800 words)
9. Case Update (300–500 words, if available)
10. Board Design
11. Appendix (if needed)
```

**Reading material category allocation:**

| Category | % | Quantity | Examples |
|----------|---|---------|---------|
| Academic frontier | 30% | 3–5 papers | Eloundou et al. (2023) GPTs are GPTs; Teece (2007) Dynamic Capabilities |
| Classic theory | 20% | 2–3 books | Rogers (2003) Diffusion of Innovations; Kotter (1996) Leading Change |
| Industry practice | 20% | 2–3 reports | McKinsey (2024) State of AI; Accenture (2022) Path to AI Maturity |
| Related cases | 15% | 2–3 cases | Comparable companies or same-issue cases from other industries |
| Supplementary | 15% | 2–3 items | Review articles, technology trend reports, policy documents |

**Reading material quality:**
- [ ] All references have complete APA citations
- [ ] Academic papers from SSCI/SCI journals
- [ ] At least 50% published in last 5 years
- [ ] Highly relevant to case discussion themes
- [ ] Cover different disciplinary perspectives
- [ ] Include both Chinese and English materials (if applicable)

**Discussion summary structure:**
1. Core insights (3–4 items): key findings, implications, theory connections
2. Theoretical contribution: how case validates or challenges existing theory
3. Practical implications: specific advice for managers
4. Teaching value: case's unique position in the course

**Discussion summary example (Pukang case):**
```
This case uses Pukang as an example to explore how intermediaries and
enterprise service providers can build evolving competitive advantages
through strategic-organizational alignment. Through discussion, students
can master:

1. General patterns of strategic evolution: from value chain to value network
2. How org structure matches strategic needs: Amoeba → Middle platform → Ecosystem
3. Three dimensions of dynamic capabilities: Sensing, Seizing, Transforming
4. New sources of competitive advantage in the digital era: from dynamic
   capabilities to data network effects

These insights have general implications for companies operating from
resource-disadvantaged, peripheral ecological positions.
```

**Supplementary materials (when possible):**
- Video: company promotional, founder interviews, product demos
- PPT: background intro (10–15 slides), theory framework lecture (15–20 slides)
- Data files: Excel financial models, industry data
- Teaching aids: role-play guides, group discussion prompt cards, student performance assessment rubric

### Good (20–22): Mostly complete structure; some summary; limited supplementary materials
### Passing (15–19): Incomplete structure; thin summary; missing readings
### Needs Improvement (0–14): Chaotic/incomplete; no summary; severely lacking
